# Introducción

La primera unidad de la asignatura **Teoría de Programación** introduce los conceptos fundamentales que sirven como base para el desarrollo de software. Durante esta unidad se estudian los principios de la lógica computacional, la representación de algoritmos mediante pseudocódigo y diagramas de flujo, así como la importancia de validar soluciones antes de implementarlas en un lenguaje de programación.

El dominio de estos contenidos permite desarrollar el pensamiento lógico y la capacidad para resolver problemas de manera estructurada, competencias esenciales para la formación de un profesional en Computación.

# Contenidos de la Unidad

En esta unidad se estudiaron los conceptos fundamentales de la programación, los cuales constituyen la base para el desarrollo de software. Antes de escribir código es necesario aprender a analizar un problema, diseñar una solución lógica y representarla mediante diferentes herramientas.

---

# 1. Algoritmo

Un **algoritmo** es un conjunto finito y ordenado de instrucciones que permiten resolver un problema o realizar una tarea específica. Todo algoritmo debe ser claro, preciso y ejecutarse en un número limitado de pasos.

### Características

- Preciso.
- Ordenado.
- Finito.
- Eficiente.
- Fácil de comprender.

### Importancia

Los algoritmos constituyen la base de cualquier programa informático, ya que permiten diseñar la solución antes de comenzar la programación.

# 2. Pseudocódigo

El **pseudocódigo** es una representación textual de un algoritmo utilizando un lenguaje estructurado similar al lenguaje humano. Su objetivo es describir la lógica del programa sin depender de la sintaxis de un lenguaje de programación específico.

### Ventajas

- Facilita el diseño de algoritmos.
- Es independiente del lenguaje de programación.
- Reduce errores antes de programar.
- Facilita el trabajo en equipo.

### Ejemplo

```text
Inicio

Leer A
Leer B

Suma ← A + B

Escribir Suma

Fin
```

# 3. Diagrama de Flujo

Un **diagrama de flujo** representa gráficamente un algoritmo mediante símbolos estandarizados conectados por flechas que indican el orden de ejecución.

## Símbolos más utilizados

| Símbolo | Significado |
|---------|-------------|
| ⭕ | Inicio / Fin |
| ▭ | Proceso |
| ◇ | Decisión |
| ▱ | Entrada / Salida |
| ➜ | Dirección del flujo |

### Ventajas

- Permite visualizar fácilmente el algoritmo.
- Facilita detectar errores.
- Mejora la comprensión del proceso.
- Sirve como documentación.

# 4. Prueba de Escritorio

La **prueba de escritorio** consiste en ejecutar manualmente un algoritmo utilizando datos de prueba para comprobar que produce los resultados esperados antes de implementarlo.

### Beneficios

- Detecta errores lógicos.
- Comprueba el funcionamiento del algoritmo.
- Reduce el tiempo de depuración.
- Facilita la comprensión del proceso.

### Ejemplo

| Variable A | Variable B | Resultado |
|-----------:|-----------:|----------:|
| 4 | 3 | 7 |
| 10 | 15 | 25 |
| 8 | 2 | 10 |

# 5. Lenguajes de Programación

Los **lenguajes de programación** permiten traducir los algoritmos a instrucciones que la computadora puede ejecutar.

## Clasificación

### Lenguaje de bajo nivel

- Cercano al hardware.
- Mayor velocidad de ejecución.
- Difícil de aprender.

**Ejemplo:** Ensamblador.

### Lenguaje de alto nivel

- Cercano al lenguaje humano.
- Fácil de aprender.
- Mayor productividad.

**Ejemplos:**

- C
- C++
- Java
- Python
- JavaScript

# 6. Programación por Bloques

La **programación por bloques** es una metodología visual que permite crear programas mediante bloques gráficos que representan instrucciones.

Este enfoque facilita el aprendizaje de la lógica de programación porque evita errores de sintaxis y permite concentrarse en el razonamiento del algoritmo.

### Ventajas

- Fácil de aprender.
- Interfaz visual.
- Favorece el pensamiento lógico.
- Ideal para principiantes.

### Plataformas conocidas

- Scratch
- Blockly
- MIT App Inventor

# Implementación en Lenguaje C

```c
#include <stdio.h>

int main() {

    float A, B, suma;

    printf("Ingrese el primer numero: ");
    scanf("%f", &A);

    printf("Ingrese el segundo numero: ");
    scanf("%f", &B);

    suma = A + B;

    printf("\nLa suma es: %.2f\n", suma);

    return 0;

}
```

---

## Explicación del Código

| Instrucción | Función |
|-------------|---------|
| `#include <stdio.h>` | Permite utilizar las funciones de entrada y salida. |
| `float A, B, suma;` | Declara las variables que almacenarán los datos. |
| `scanf()` | Lee los números ingresados por el usuario. |
| `suma = A + B;` | Realiza la operación matemática. |
| `printf()` | Muestra el resultado en pantalla. |
| `return 0;` | Finaliza correctamente el programa. |

# Reflexión Crítica

## Dificultades Encontradas

Durante el desarrollo de esta unidad, una de las principales dificultades fue comprender la importancia de planificar la solución de un problema antes de comenzar a programar. Al inicio resultó complejo diferenciar las funciones del algoritmo, el pseudocódigo y el diagrama de flujo, ya que cada herramienta representa la solución desde una perspectiva distinta.

Otra dificultad consistió en interpretar correctamente la secuencia lógica de las instrucciones y evitar errores al momento de trasladar el algoritmo al lenguaje de programación C.

---

## Estrategias Aplicadas

Para superar estas dificultades se realizaron las siguientes actividades:

- Análisis paso a paso de cada problema.
- Elaboración de algoritmos antes de escribir el código.
- Desarrollo de pseudocódigos utilizando PSeInt.
- Diseño de diagramas de flujo para visualizar la lógica del programa.
- Ejecución de pruebas de escritorio para validar los resultados.
- Implementación del algoritmo en lenguaje C y comprobación de su funcionamiento.

Estas estrategias permitieron comprender mejor la lógica de programación y reducir errores durante la codificación.

---

Al finalizar la unidad se fortalecieron conocimientos relacionados con:
